## 3 à 5 recos prioritaires

1. **Compresser la meta desc à 150-158 chars** : "Andrej Karpathy, ex-OpenAI et ex-Tesla, est le meilleur prof d'IA. Je te résume ses 3 idées clés et ses 4 ressources — même si tu n'es pas dev." (148 chars, conserve crédibilité + bénéfice). Supprimer le suffixe "Par Jérémy Sagnier" qui n'apporte rien à 173 chars.

2. **Ajouter 3-4 illustrations originales** : un schéma Software 1.0 / 2.0 / 3.0 (l'idée signature de Karpathy), une visualisation du tokenizer (texte → tokens), un schéma du LLM OS, une timeline de sa carrière (OpenAI → Tesla → OpenAI → Eureka). Génération possible en 1 prompt Claude Code via SVG inline. Manque flagrant pour un article 2200 mots qui se passe d'image — pénalisant pour Google Images, dwell time, et partage social (l'OG affiche quoi sans `og:image` ?).

3. **Aligner les `datePublished` dans les 2 JSON-LD ET fusionner les schemas** : actuellement Article = 2026-04-08, TechArticle = 2026-04-20. Garder un seul schema (Article suffit pour ce contenu, TechArticle est plus adapté à une doc technique avec dependencies/proficiencyLevel). Date = 8 avril (publi originale), modified = 20 avril.

4. **Renommer le slug en `karpathy-vulgarise.html` ou `andrej-karpathy-explique.html`** (avec redirect 301 depuis `karpathy.html`). Le slug actuel `karpathy` est ultra-fort en brand match mais zéro intent. Avec "vulgarise" ou "explique" tu captes l'intent informationnel.

5. **Ajouter un bloc "Transparence IA"** en bas (ou dans la byline) : "Cet article est rédigé par Jérémy Sagnier (humain) avec l'aide de Claude (vérification factuelle, structuration). Toutes les sources YouTube/X sont vérifiées et linkées." — alignement direct avec le pitch éditorial du site CLAUDE.md (ligne "Hyper transparent : assumer l'usage de l'IA").
