## 3 à 5 recos prioritaires (actionnables)

1. **Raccourcir le title à ≤ 60 chars** : ex. "Limova 840 €/an : faut-il vraiment payer ? Mon analyse" (53 chars) ou "Limova vs Claude Code : 840 € vs 30 € par mois" (47 chars). Vire les guillemets français + le "par Jérémy Sagnier" redondant.
2. **Raccourcir la meta description à 150-160 chars** : ex. "Limova : 9 agents IA préemballés à 840 €/an. J'ai creusé l'offre. Voici ce qui m'a attiré, ce qui m'a fait tiquer, et l'alternative DIY à 30 €/mois." (159 chars).
3. **Ajouter 5-8 liens externes essentiels** : limova.ai (page tarifs), Trustpilot Limova, journal-economique.fr/article-cité (avec URL réelle), G2/Capterra (pour montrer l'absence d'avis comme preuve), Anthropic Claude Code, Twilio, Deepgram, ElevenLabs (cités comme alternatives DIY). Sans liens, les affirmations sourcées de l'article perdent toute valeur EEAT — pour un article qui *attaque* un concurrent, c'est juridiquement et éditorialement risqué.
4. **Migrer le JSON-LD vers `Review` avec `itemReviewed: SoftwareApplication` Limova + `reviewRating`** — gain rich snippet étoiles dans les SERP, taux de clic boosté pour ce type de contenu comparatif. ROI très élevé.
5. **Ajouter OG image custom** (réutiliser une des 3 captures Limova en cover, ou créer un visuel "840 € vs 30 €" choc) — pour le partage LinkedIn de cet article qui est ultra-shareable par nature comparatif.
